# S3 credentials <a href="#s3-credentials" id="s3-credentials"></a>

你可以使用这些 credentials 对以下 nodes 进行身份验证：

- [S3](../app-nodes/n8n-nodes-base.s3.md)

## 前提条件 <a href="#prerequisites" id="prerequisites"></a>

在兼容 S3 的 server 上创建一个 account。S3 node 可用于通用或非 AWS S3 服务，例如：

* [DigitalOcean Spaces](https://www.digitalocean.com/products/spaces)
* [MinIO](https://min.io/)
* [Tigris](https://www.tigrisdata.com/)
* [Wasabi](https://wasabi.com/)

## 支持的身份验证方式 <a href="#supported-authentication-methods" id="supported-authentication-methods"></a>

- S3 endpoint

## 相关资源 <a href="#related-resources" id="related-resources"></a>

有关服务的更多信息，请参考你的 S3-compatible provider 文档。例如，请参考 [Wasabi's REST API documentation](https://docs.wasabi.com/docs/rest-api-introduction) 或 [DigitalOcean's Spaces API Reference Documentation](https://docs.digitalocean.com/reference/api/spaces-api/)。

## 使用 S3 endpoint <a href="#using-s3-endpoint" id="using-s3-endpoint"></a>

要配置此 credential，你需要：

- 一个 **S3 Endpoint**：输入 S3 storage backend 的 URL endpoint。
- 一个 **Region**：输入你的 S3 storage 所在 region。有些 providers 称之为 "region slug"。
- 一个 **Access Key ID**：输入 S3 provider 用于访问 bucket 或 space 的 S3 access key。有些 providers 称之为 API keys。
- 一个 **Secret Access Key**：输入 **Access Key ID** 对应的 secret access key。
- **Force Path Style**：启用后，connection 会对 buckets 使用 path-style addressing。
- **Ignore SSL Issues**：启用后，即使 SSL certificate validation 失败，n8n 也会继续连接。

下面是 DigitalOcean Spaces、Tigris 和 Wasabi 的更详细说明。如果你使用其他 provider，请参考其文档了解更多信息。

### 使用 DigitalOcean Spaces <a href="#using-digitalocean-spaces" id="using-digitalocean-spaces"></a>

配置用于 DigitalOcean Spaces 的 credential：

1. 在 DigitalOcean 中，前往 control panel 并打开 **Settings**。你的 endpoint 应列在那里。在该 endpoint 前加上 `https://`，并在 n8n 中将其输入为 **S3 Endpoint**。
    - 你的 DigitalOcean endpoint 取决于 bucket 所在的数据中心 region。
2. 对于 **Region**，输入 bucket 所在的 region，例如 `nyc3`。
    - 如果你计划使用此 credential 创建新的 Spaces，请改为输入 `us-east-1`。
3. 从 DigitalOcean control panel 前往 [**API**](https://cloud.digitalocean.com/account/api/spaces)。
4. 打开 **Spaces Keys** tab。
5. 选择 **Generate New Key**。
6. 为你的 key 输入 **Name**，例如 `n8n integration`，并选择 checkmark。
7. 复制名称旁边显示的 **Key**，并在 n8n 中将其输入为 **Access Key ID**。
8. 复制 **Secret** value，并在 n8n 中将其输入为 **Secret Access Key**。
    - 有关生成 key 和 secret 的更多信息，请参考 [Sharing Access to Buckets with Access Keys](https://docs.digitalocean.com/products/spaces/how-to/manage-access/#access-keys)。
9. 保持 **Force Path Style** toggle 关闭，除非你想使用 subdomain/virtual calling format。
10. 决定 n8n credential 如何处理 SSL：
    - 要遵守 SSL certificate validation，请保持默认的 **Ignore SSL Issues** 关闭。
    - 要在 SSL certificate validation 失败时仍连接，请打开 **Ignore SSL Issues**。

更多信息请参考 DigitalOcean 的 [Spaces API Reference Documentation](https://docs.digitalocean.com/reference/api/spaces-api/)。

### 使用 Tigris <a href="#using-tigris" id="using-tigris"></a>

配置用于 Tigris 的 credential：

1. 对于 **S3 Endpoint**，输入 `https://t3.storage.dev`。
    - Tigris 使用单个 global endpoint，不需要选择 region；data 会从离 request 最近的 region 提供。
2. 对于 **Region**，输入 `auto`。
    - 该值是 SigV4 request signing 所必需的，但 Tigris service 会忽略它。
3. 登录 [Tigris Console](https://console.storage.dev/)。
4. 从 sidebar 选择 **Access Keys**。
5. 在右上角选择 **Create New Access Key**。
6. 为 key 输入 **Name**，例如 `n8n integration`，并选择 **Create**。
7. 复制显示的 **Access Key ID**（以 `tid_` 开头），并在 n8n 中将其输入为 **Access Key ID**。
8. 复制显示的 **Secret Access Key**（以 `tsec_` 开头），并在 n8n 中将其输入为 **Secret Access Key**。
    - secret 只显示一次。如果丢失，需要创建新的 key。
9. 保持 **Force Path Style** toggle 关闭。Tigris 使用 virtual-hosted-style addressing。
10. 决定 n8n credential 如何处理 SSL：
    - 要遵守 SSL certificate validation，请保持默认的 **Ignore SSL Issues** 关闭。
    - 要在 SSL certificate validation 失败时仍连接，请打开 **Ignore SSL Issues**。

更多信息请参考 Tigris 的 [Manage Access Keys documentation](https://www.tigrisdata.com/docs/iam/manage-access-key/)。

### 使用 Wasabi <a href="#using-wasabi" id="using-wasabi"></a>

配置用于 Wasabi 的 credential：

1. 对于 **S3 Endpoint**，输入 bucket region 的 service URL。URL 需要以 `https://` 开头。
    - 请参考 [Service URLs for Wasabi's Storage Regions](https://docs.wasabi.com/v1/docs/service-urls-for-wasabis-storage-regions) 识别正确 URL。
2. 对于 **Region**，输入 service URL 中的 region slug 部分。例如，如果你将 `https://s3.us-east-2.wasabisys.com` 输入为 **S3 Endpoint**，则 `us-east-2` 就是 region。
3. 以 root user 登录 Wasabi Console。
4. 打开 **Menu** 并选择 **Access Keys**。
5. 选择 **CREATE NEW ACCESS KEY**。
6. 选择该 key 是用于 **Root User** 还是 **Sub-User**，并选择 **CREATE**。
7. 复制 **Access Key**，并在 n8n 中将其输入为 **Access Key ID**。
8. 复制 **Secret Key**，并在 n8n 中将其输入为 **Secret Access Key**。
    - 有关生成 key 和 secret 的更多信息，请参考 [Creating a New Access Key](https://docs.wasabi.com/v1/docs/creating-a-new-access-key)。
9. Wasabi 建议打开 **Force Path Style** toggle，“because the path-style offers the greatest flexibility in bucket names, avoiding domain name issues.” 更多信息请参考 Wasabi [REST API Introduction](https://docs.wasabi.com/docs/rest-api-introduction)。
10. 决定 n8n credential 如何处理 SSL：
    - 要遵守 SSL certificate validation，请保持默认的 **Ignore SSL Issues** 关闭。
    - 要在 SSL certificate validation 失败时仍连接，请打开 **Ignore SSL Issues**。
