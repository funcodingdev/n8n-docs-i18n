## 设置 node <a href="#setting-up-the-node" id="setting-up-the-node"></a>

### 使用 query parameter 设置默认选项 <a href="#set-default-selections-with-query-parameters" id="set-default-selections-with-query-parameters"></a>

你可以在 [n8n Form Trigger node](n8n-nodes-base.formtrigger.md) 提供的初始 URL 上使用 [query parameter](https://en.wikipedia.org/wiki/Query_string#Web_forms)，为字段设置初始值。表单中的每个页面都会收到发送到 n8n Form Trigger node URL 的相同 query parameter。

{% hint style="info" %}
**仅限生产环境**

Query parameter 仅在生产模式下使用表单时可用。测试模式下，n8n 不会根据 query parameter 填充字段值。
{% endhint %}


使用 query parameter 时，请对任何包含特殊字符的字段名或值进行 [percent-encode](https://en.wikipedia.org/wiki/Percent-encoding)。这可以确保 n8n 使用给定字段的初始值。你可以使用 [URL Encode/Decode](https://www.url-encode-decode.com/) 等工具，用 percent-encoding 格式化 query parameter。

例如，假设你有一个包含以下属性的表单：

* Production URL: `https://my-account.n8n.cloud/form/my-form`
* Fields:
	* `name`: `Jane Doe`
	* `email`: `jane.doe@example.com`

通过 query parameter 和 percent-encoding，可以使用以下 URL 将初始字段值设置为上面的数据：

```
https://my-account.n8n.cloud/form/my-form?email=jane.doe%40example.com&name=Jane%20Doe
```

这里，percent-encoding 会将 at 符号（`@`）替换为字符串 `%40`，并将空格字符（` `）替换为字符串 `%20`。无论这些字段出现在表单的哪个页面上，都会设置它们的初始值。


### 显示自定义 HTML <a href="#displaying-custom-html" id="displaying-custom-html"></a>

你可以通过向表单添加 **Custom HTML** 字段，在表单上显示自定义 HTML。它会提供一个 **HTML** 文本框，你可以在其中插入任意 HTML 代码，作为表单页面的一部分显示。

你可以在 HTML 字段中加入链接、图片、视频等内容，丰富表单页面。n8n 会在正常文档流中将这些内容与其他表单字段一起渲染。

由于自定义 HTML 内容是只读的，默认情况下这些字段不会包含在表单输出数据中。要在 node 输出中包含原始 HTML 内容，请使用 **Element Name** 字段为该数据提供名称。

HTML 字段不支持 `<script>`、`<style>` 或 `<input>` 元素。

如果使用 [Form Ending](#form-ending) Page Type，可以将 **On n8n Form Submission** 参数选择为 **Show Text**，从而完全自定义发送给用户的最终页面（包括使用 `<script>`、`<style>` 和 `<input>` 元素）。

### 包含隐藏字段 <a href="#including-hidden-fields" id="including-hidden-fields"></a>

你可以在表单中包含不会显示给用户的字段。当你想向表单传递不需要用户交互输入的额外数据时，这很有用。

要添加不会显示在表单上的字段，请使用 **Hidden Field** 表单元素。你可以在其中定义 **Field Name**，并可选择通过填写 **Field Value** 提供默认值。

提供表单时，你可以使用 [query parameter](#set-default-selections-with-query-parameters) 为隐藏字段传值。

### 使用 JSON 定义表单 <a href="#defining-the-form-using-json" id="defining-the-form-using-json"></a>

使用 **Define Form** > **Using JSON**，通过[对象组成的 JSON 数组](https://app.gitbook.com/s/rPN1zU5jaYNvwH7RzxqA/work-with-data/understand-n8ns-data-structure)定义表单字段。每个对象通过这些 key 的组合定义单个字段：

- `fieldLabel`：渲染后的表单上显示在输入字段上方的标签。
- `fieldName`：用于 Form node 输出，以及在 expression 中引用该字段的字段名。
- `fieldType`：可选择 `checkbox`、`date`、`dropdown`、`email`、`file`、`hiddenField`、`html`、`number`、`password`、`radio`、`text` 或 `textarea`。
    - 使用 `date` 可在表单中包含日期选择器。有关日期格式化的更多信息，请参阅 [Date and time with Luxon](https://app.gitbook.com/s/rPN1zU5jaYNvwH7RzxqA/work-with-data/handle-special-data-types/work-with-dates-and-times)。
	- 使用 `dropdown` 时，通过 `fieldOptions` 设置选项（参考下面的示例）。默认情况下，下拉框是单选。要改为多选，请将 `multiselect` 设置为 `true`。作为替代，也可以将 `checkbox` 或 `radio` 与 `fieldOptions` 一起使用。
	- 使用 `file` 时，将 `multipleFiles` 设置为 `true`，以允许用户选择多个文件。要定义允许的文件类型，请将 `acceptFileTypes` 设置为包含逗号分隔文件扩展名列表的字符串（参考下面的示例）。
	- 使用 `hiddenField` 可向表单添加隐藏字段。更多信息请参阅[包含隐藏字段](#including-hidden-fields)。
	- 使用 `html` 可在表单中显示自定义 HTML。更多信息请参阅[显示自定义 HTML](#displaying-custom-html)。
- `placeholder`：指定字段的 placeholder 数据。除 `dropdown`、`checkbox`、`radio`、`date` 和 `file` 以外，可用于所有 `fieldType`。
- `defaultValue`：指定将在表单元素中预填充或预选的值。除 `password`、`html`、`hiddenField` 和 `file` 以外，可用于所有 `fieldType`。
- `requiredField`：要求用户在表单中完成此字段。

以下 JSON 示例展示了所需的一般格式和可用 key：

```javascript
// Use the "requiredField" key on any field to mark it as mandatory
// Use the "placeholder" key to specify placeholder data for all fields
// except 'dropdown', 'checkbox', 'radio', 'date' and 'file'
// Use the "defaultValue" key to pre-fill a form field or pre-select a
// value in 'checkbox', 'radio' or 'dropdown' fields

[
  {
    "fieldLabel": "Date Field",
    "fieldType": "date",
    "formatDate": "mm/dd/yyyy", // how to format received date in n8n
    "requiredField": true
  },
  {
    "fieldLabel": "Dropdown Options",
    "fieldType": "dropdown",
    "fieldOptions": {
      "values": [
        {
          "option": "option 1"
        },
        {
          "option": "option 2"
        }
      ]
    },
    "defaultValue": "option 1",
    "requiredField": true
  },
  {
    "fieldLabel": "Multiselect",
    "fieldType": "dropdown",
    "fieldOptions": {
      "values": [
        {
          "option": "option 1"
        },
        {
          "option": "option 2"
        }
      ]
    },
    "multiselect": true // setting to true allows multi-select
  },
  {
    "fieldLabel": "Email",
    "fieldType": "email",
    "placeholder": "me@mail.com"
  },
  {
    "fieldLabel": "File",
    "fieldType": "file",
    "multipleFiles": true, // setting to true allows multiple files selection
    "acceptFileTypes": ".jpg, .png" // allowed file types
  },
  {
    "fieldLabel": "Number",
    "fieldType": "number"
  },
  {
    "fieldLabel": "Password",
    "fieldType": "password"
  },
  {
    // "fieldType": "text" can be omitted since it's the default type
    "fieldLabel": "Text"
  },
  {
    "fieldLabel": "Textarea",
    "fieldType": "textarea",
    "defaultValue": "Lorem ipsum."
  },
  {
    "fieldType": "html",
    "elementName": "content", // Optional field. It can be used to include the html in the output.
    "html": "<div>Custom element</div>"
  },
  {
    "fieldLabel": "Checkboxes",
    "fieldType": "checkbox",
    "fieldOptions": {
      "values": [
        {
          "option": "option 1"
        },
        {
          "option": "option 2"
        }
      ]
    },
    "defaultValue": ["option 1", "option 2"]
  },
  {
    "fieldLabel": "Radio",
    "fieldType": "radio",
    "fieldOptions": {
      "values": [
        {
          "option": "option 1"
        },
        {
          "option": "option 2"
        }
      ]
    }
  },
  {
    "fieldLabel": "hidden label",
    "fieldType": "hiddenField",
    "fieldValue": "extra form data"
  }
]
```

### Form Ending <a href="#form-ending" id="form-ending"></a>

使用 **Form Ending** Page Type 可以结束表单，并显示完成页面、将用户重定向到 URL，或显示自定义 HTML 或文本。即使 n8n 执行包含 Form Ending node 的[多个分支](#forms-with-branches)，每次 execution 也只会显示一个 Form Ending 页面。

使用 **On n8n Form Submission** 时，可在这些选项之间选择：

- **Show Completion Screen**：向用户显示最终页面，确认他们已提交表单。
	- 填写 **Completion Title**，设置表单上的 `h1` 标题。
	- n8n 会将 **Completion Message** 显示为表单主 `h1` 标题下方的副标题。使用 `\n` 或 `<br>` 添加换行。
	- 选择 **Add option** 并填写 **Completion Page Title**，设置浏览器标签页中的页面标题。
- **Redirect to URL**：表单完成时将用户重定向到指定 URL。
	- 在 **URL** 字段中填写用户完成表单后要重定向到的页面。
- **Show Text**：显示由任意纯文本和 HTML 定义的最终页面。
	- 在 **Text** 字段中填写要显示的 HTML 或纯文本内容。
- **Return Binary File**：完成后返回二进制文件。
	- 填写 **Completion Title**，设置表单上的 `h1` 标题。
	- n8n 会将 **Completion Message** 显示为表单主 `h1` 标题下方的副标题。使用 `\n` 或 `<br>` 添加换行。
	- 提供包含要返回给用户的二进制文件的 **Input Data Field Name**。

### 带分支的表单 <a href="#forms-with-branches" id="forms-with-branches"></a>

n8n Form node 每当从前一个 node 收到数据时，都会执行并显示其关联的表单页面。在使用 n8n 构建表单时，为避免困惑，理解出现分支时表单的行为很重要。

#### 具有互斥分支的 workflow <a href="#workflows-with-mutually-exclusive-branches" id="workflows-with-mutually-exclusive-branches"></a>

包含互斥分支的表单 workflow 会按预期工作。n8n 会根据提交的数据和你定义的条件执行单个分支。执行过程中，n8n 会显示该分支中的每个页面，并以 **Form Ending** page type 的 n8n Form node 结束。

此 workflow 演示互斥分支。每个选择只能执行单个分支。

#### 可能执行多个分支的 workflow <a href="#workflows-that-may-execute-multiple-branches" id="workflows-that-may-execute-multiple-branches"></a>

如果表单 workflow 会同时向多个分支发送数据，就需要更谨慎。当一次 execution 中多个分支收到数据时（例如来自 [switch](n8n-nodes-base.switch.md) node），n8n 会[按顺序](https://app.gitbook.com/s/rPN1zU5jaYNvwH7RzxqA/flow-logic/understand-execution-order)执行每个收到数据的分支。到达一个分支末尾后，execution 会移动到下一个有数据的分支。

每次 execution 中，n8n 只会执行一个 **Form Ending** n8n Form node。当表单 workflow 的多个分支收到数据时，除了与最终分支关联的 Form Ending node 以外，n8n 会忽略所有其他 Form Ending node。

此 workflow 在一次 execution 中可能会执行多个分支。在这里，n8n 会按顺序执行所有有效分支。这会影响 n8n 执行哪些 n8n Form node，尤其会影响显示哪个 **Form Ending** node：

### Node 选项 <a href="#node-options" id="node-options"></a>

选择 **Add Option** 可查看更多配置选项：

- **Form Title**：表单标题。n8n 会将 **Form Title** 显示为网页标题和表单上的主 `h1` 标题。
- **Form Description**：表单描述。n8n 会将 **Form Description** 显示为表单主 `h1` 标题下方的副标题。此字段支持 HTML。使用 `\n` 或 `<br>` 添加换行。Form Description 还会填充页面的 [HTML meta description](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/meta/name#standard_metadata_names_defined_in_the_html_specification)。
- **Button Label**：表单提交按钮使用的标签。n8n 会将 **Button Label** 显示为提交按钮名称。
- **Custom Form Styling**：使用 CSS 覆盖公开表单界面的默认样式。该字段会预填充默认样式，因此你只需修改需要的部分。
- **Completion Page Title**：表单最终完成页面的标题。

### 构建和测试 workflow <a href="#build-and-test-workflows" id="build-and-test-workflows"></a>

构建或测试 workflow 时，请使用 [n8n Form Trigger node](n8n-nodes-base.formtrigger.md) 中的 **Test URL**。使用 test URL 可以确保你能在编辑器 UI 中查看传入数据，这对调试很有用。

有两种测试方式：

- 选择 **Execute Step**。n8n 会打开表单。提交表单后，n8n 会运行该 node 和任何之前的 node，但不会运行 workflow 的其余部分。
- 选择 **Execute Workflow**。n8n 会打开表单。提交表单后，n8n 会运行整个 workflow。

### 生产 workflow <a href="#production-workflows" id="production-workflows"></a>

当 workflow 准备就绪后，打开 trigger node，并在 **From URLS** 选择器中选择 **Production URL**，切换为使用 n8n Form Trigger 的 **Production URL**。然后你可以发布 workflow，用户提交表单时 n8n 会自动运行它。

使用 production URL 时，请确保已保存并发布 workflow。通过 Form trigger 流动的数据在使用 production URL 时不会显示在编辑器 UI 中。
