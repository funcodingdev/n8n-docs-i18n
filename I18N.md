# n8n docs i18n architecture

This fork keeps upstream synchronization simple by treating English as the upstream-owned source and Simplified Chinese as an overlay.

## Directory layout

- `docs/`: upstream English documentation. Keep this as close to upstream as possible.
- `docs/zh/`: Simplified Chinese translations. Mirror the English relative path when a page is translated.
- `mkdocs.yml`: shared site configuration and top-header language metadata.
- `_overrides/partials/header.html`: shared header UI. The language selector maps the current page URL to its mirrored language path.

Example:

```text
docs/get-started/learning-paths.md
docs/zh/get-started/learning-paths.md
```

## Language switch behavior

The header selector assumes:

- English pages are served from the site root.
- Simplified Chinese pages are served under `/zh/`.
- A translated page uses the same path under `docs/zh/` as its English source uses under `docs/`.
- The active translated language code and prefix live in `extra.i18n` in `mkdocs.yml`.

If a mirrored page does not exist yet, the generated link may 404. Translate pages by creating the mirrored file before publishing a public link to that page.

## Upstream sync workflow

1. Pull or merge upstream changes into the fork.
2. Resolve conflicts in upstream-owned files first, especially `docs/`, `nav.yml`, and `_overrides/`.
3. Keep local i18n changes small and isolated:
   - `docs/zh/**`
   - `I18N.md`
   - the `extra.i18n` and `extra.alternate` block in `mkdocs.yml`
   - the language selector block in `_overrides/partials/header.html`
4. For changed English pages, update the matching Chinese page under `docs/zh/`.

## Future upgrade path

When the Chinese coverage is large enough, consider adding `mkdocs-static-i18n` and moving language-specific navigation labels into plugin configuration. Until then, this overlay layout avoids moving upstream files and keeps merge conflicts lower.
